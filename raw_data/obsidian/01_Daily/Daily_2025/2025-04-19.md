1년 뒤에도 정보를 효과적으로 찾고 활용하기 위한 몇 가지 아이디어를 제안합니다.

1. **일관된 링크 사용:** 노트 간의 의미 있는 연결 (`[[ ]]`)을 꾸준히 만드세요. 관련 프로젝트, Task, Daily Note, 개념들을 서로 연결하면 정보 탐색이 용이해집니다. Backlink 활용도 중요합니다.
2. **명확한 태그 시스템:** `tags:` 필드에 일관된 규칙으로 태그를 부여하세요 (예: `#project/ProjectName`, `#status/doing`, `#topic/AI`). 검색 및 Dataview 쿼리에 유용합니다.
3. **간결한 폴더 구조:** 너무 복잡하지 않게 `01_Projects`, `02_Areas`, `03_Resources`, `04_Archive`, `05_Daily` 등 큰 범주로 나누는 것을 고려해보세요. 중요한 것은 링크와 태그입니다.
4. **핵심 메타데이터 활용:** 앞서 제안된 `type`, `status`, `date`, `project`, `epic` 등의 메타데이터를 꾸준히 작성하면 Dataview를 통해 원하는 정보를 정확히 필터링하고 집계할 수 있습니다.
5. **주기적인 리뷰 및 리팩토링:** 주간/월간 리뷰를 통해 노트를 검토하고, 중복 내용을 합치거나, 오래된 정보는 아카이빙하며, 링크와 태그를 최신 상태로 유지하세요.
6. **MOC (Maps of Content) 활용:** 특정 주제나 프로젝트에 대한 핵심 노트들을 연결하는 목차(MOC) 노트를 만드세요. 관련 정보로 빠르게 이동하는 허브 역할을 합니다.
7. **템플릿 적극 활용:** 노트 유형별 (Task, Project, Daily Note 등) 템플릿을 만들어 메타데이터와 기본 구조를 통일하면 일관성 유지에 도움이 됩니다.

이 방법들을 조합하면 시간이 지나도 필요한 정보를 쉽게 찾고 연결된 맥락 속에서 활용할 수 있을 것입니다.

태그를 `#project/ProjectName`과 같이 계층적으로 사용하는 방식(Nested Tags)의 주요 장점은 다음과 같습니다.

1. **구조화 및 명확성:** 태그에 `project`라는 상위 분류를 부여하여 해당 태그가 프로젝트와 관련 있음을 명확히 알 수 있습니다. `#ProjectName`만 사용하는 것보다 의미가 분명해집니다.
2. **효율적인 필터링:**
    - `#project` 태그만 검색하면 모든 프로젝트 관련 노트를 한 번에 볼 수 있습니다.
    - `#project/ProjectName`을 검색하면 특정 프로젝트 관련 노트만 정확히 필터링할 수 있습니다.
3. **Dataview 활용 용이:** Dataview 쿼리에서 `contains(file.tags, "#project")`와 같이 상위 태그를 기준으로 필터링하거나, 특정 프로젝트 태그를 기준으로 필터링하기 편리합니다.
4. **확장성:** 프로젝트 수가 늘어나도 `#project`라는 일관된 분류 체계 안에서 관리되므로 태그 시스템이 복잡해지는 것을 방지할 수 있습니다. 다른 분류(예: `#status/done`, `#topic/AI`)와 혼동될 가능성도 줄어듭니다.

네, 맞습니다. Epic과 Task에도 동일한 계층적 태그 방식 (`#epic/EpicName`, `#task/TaskIdentifier`)을 적용하면 유사한 장점을 얻을 수 있습니다.

- **구조화:** `#epic`이나 `#task`라는 상위 분류를 통해 태그의 성격을 명확히 할 수 있습니다.
- **필터링:** 모든 Epic 관련 노트를 보려면 `#epic`을, 특정 Epic만 보려면 `#epic/EpicName`을 검색/필터링할 수 있습니다. Task도 마찬가지입니다.
- **Dataview 활용:** 상위 태그(`epic`, `task`)나 특정 하위 태그를 기준으로 Dataview 쿼리를 작성하기 용이합니다.
- **확장성:** Epic이나 Task가 많아져도 일관된 분류 체계 내에서 관리할 수 있습니다.

---

1. **간결함 유지:** 꼭 필요한 정보만 메타데이터에 포함하세요. 너무 많은 필드는 관리하기 어렵고 노트 작성을 번거롭게 만듭니다. 처음에는 핵심 필드(`type`, `tags`, `date` 등)로 시작하고 필요에 따라 추가하세요.
2. **일관성 확보:** 필드 이름과 값의 형식을 일관되게 사용하세요 (예: 날짜는 항상 `YYYY-MM-DD` 형식). 템플릿 플러그인(Templates, Templater)을 사용하면 일관성을 유지하는 데 큰 도움이 됩니다.
3. **목적 의식:** 실제로 검색, 필터링(Dataview 등), 분류에 사용할 메타데이터만 추가하세요. '혹시 필요할까 봐' 넣는 필드는 최소화하는 것이 좋습니다.
4. **링크 vs. 태그 vs. 필드 구분:**
    - **링크 (`[[Note Name]]`):** 특정 노트 간의 직접적인 관계를 나타낼 때 사용합니다 (예: `project: [[프로젝트 A]]`).
    - **태그 (`#tag`):** 상태, 주제, 우선순위 등 광범위한 분류에 사용합니다 (예: `#status/doing`, `#topic/AI`, `#priority/high`). 계층적 태그는 구조화에 유용합니다.
    - **필드 (`key: value`):** 날짜, 숫자, 특정 텍스트 값 등 구조화된 데이터를 저장할 때 사용합니다 (예: `due_date: 2024-12-31`, `담당자: 홍길동`). Dataview 계산이나 특정 값 조회에 유용합니다.
5. **날짜 활용:** `creation_date`, `due_date`, `start_date` 등 필요한 날짜 필드를 일관된 형식(`YYYY-MM-DD`)으로 사용하면 정렬 및 검색이 용이합니다.
6. **Dataview 고려:** 메타데이터를 설계할 때 Dataview로 어떻게 활용할지 미리 생각해보면 좋습니다. 어떤 정보를 필터링하고, 목록화하고, 집계할지에 따라 필요한 필드가 달라집니다.
7. **정기적 검토:** 가끔씩 자신의 메타데이터 사용 방식을 검토하고 개선하세요. 사용하지 않는 필드는 제거하고, 필요한 필드는 추가하며 시스템을 발전시키세요.

### ✅ 기능별 설계 개요

#### 📅 Daily Report
- 날짜별 Task 로그 자동 포함
- 시간, 에너지 소비량 집계
- 요약 자동 생성 (AI 이용)
- 주간 Report 생성 시 하루 단위 리포트 자동 합산

#### 🧩 Task
- 생성 시 QuickAdd로 생성 (Template 활용)
- Task마다:
  - `Project`, `Epic`, `Domain(Career, Mind 등)` 메타데이터 입력
  - `Time Log`, `Energy Level`, `Status`, `Due Date` 입력 가능
  - Unique ID 생성 (`{{date}}-{{time}}` 등)
  - 백링크로 Project / Epic 페이지 자동 연결
  - 타임 로그에 따라 Daily Report 자동 연결

#### 🏗️ Project / Epic
- 개별 페이지 존재
- 관련 Task 들 백링크로 자동 정리 (`[[Task Note]]`)
- 시각적 트리 구조는 Excalibrain 활용
- 시간 흐름 / 에너지 흐름 / 문제점 등 리포트 영역 구성
- 요약 작성용 템플릿 구성 (리뷰 가능)

#### ⚙️ 스크립트 예시
- QuickAdd를 이용해 Task 생성 시:
  - 자동으로 현재 시간 기반 Note 생성
  - 태그/메타데이터 자동 입력
  - Project, Epic 명칭 입력 받아 템플릿에 삽입
- DataviewJS로:
  - Daily Report 내 Task 시간 합산
  - Weekly Report로 묶기
  - 특정 Project/Epic/Domain 내 Task 모아서 총 시간/에너지 보기
