### Lecture
 
제프 베조스는 영상 후반부에서 현재의 AI 열풍을 2000년대 초반의 **인터넷 버블**에 비유하며 다음과 같은 인사이트를 공유했습니다.
### **1. 현재 AI 분야는 '버블'과 유사하다.**

- **닷컴 버블과의 유사점**: 베조스는 현재 AI에 대한 엄청난 낙관론과 흥분, 그리고 막대한 투자가 몰리는 상황이 닷컴 버블 당시와 매우 유사하다고 말합니다.
    
- **좋은 아이디어와 나쁜 아이디어의 혼재**: 이러한 시기에는 모든 아이디어(좋은 것과 나쁜 것 모두)가 투자를 받기 쉽습니다. 투자자들조차 둘을 구분하기 어려워하며, 이로 인해 많은 AI 스타트업이 생겨나고 있습니다.
    
- **주가와 기업의 본질은 다를 수 있다**: 닷컴 버블이 터졌을 때 아마존의 주가는 113달러에서 6달러로 폭락했지만, 실제 고객 수, 매출, 이익률 등 모든 사업 지표는 꾸준히 성장하고 있었습니다. 그는 이를 근거로 현재 AI 기업들의 높은 가치 평가와 실제 사업의 본질은 분리해서 봐야 한다고 강조합니다.
### **2. AI는 모든 산업을 바꿀 '수평적 기술'이다.**

- **특정 분야 기술이 아니다**: 베조스는 AI를 특정 회사나 산업에 국한된 기술이 아니라, 모든 산업에 수평적으로 적용되어 생산성과 품질을 높이는 **'수평적 활성화 계층(horizontal enabling layer)'**이라고 정의합니다.
    
- **모든 기업에 미치는 영향**: 제조업, 호텔, 소비재 등 세상의 모든 회사가 AI의 영향을 받아 변화할 것이라고 예측합니다. 이는 AI가 단순히 새로운 산업을 만드는 것을 넘어, 기존의 모든 산업을 근본적으로 혁신할 것임을 의미합니다.
### **3. '산업 버블'은 사회에 긍정적일 수 있다.**

- **금융 버블과의 차이점**: 베조스는 2008년 금융 위기 같은 '금융 버블'은 사회에 해롭기만 하지만, 닷컴 버블이나 현재의 AI 열풍과 같은 **'산업 버블(industrial bubble)'**은 다르다고 설명합니다.
    
- **사회에 남는 유산**: 닷컴 버블 시기에 수많은 인터넷 기업들이 파산했지만, 그들이 깔아놓은 광케이블과 같은 **인프라**는 남아서 사회 전체가 그 혜택을 누렸습니다. 마찬가지로, 현재 많은 AI 기업들이 생겨나고 사라지더라도 그 과정에서 개발된 기술과 인프라는 남아 사회에 거대한 이익을 가져다줄 것이라고 봅니다.
    
- **결국 사회는 승리한다**: 비록 많은 투자자들이 돈을 잃을 수 있지만, 버블이 끝난 후 살아남은 혁신적인 기술과 기업들로 인해 사회 전체는 발전하고 큰 혜택을 얻게 될 것이라고 낙관합니다.

결론적으로 제프 베조스는 현재의 AI 열풍에 버블과 같은 위험 요소가 있는 것은 사실이지만, AI 기술 자체가 가진 잠재력은 진짜이며 장기적으로 모든 산업과 사회에 엄청난 긍정적 변화를 가져올 것이라고 확신하고 있습니다.

    // 테이블 데이터 생성
    const tableData = [];
    let totalDayMinutes = 0;

    // 해당 날짜의 총 시간 계산
    for (const activity of activityNames) {
      totalDayMinutes += dailyActivities[activity];
    }

    // 각 활동별 데이터 생성
    for (const activity of activityNames) {
      const duration = dailyActivities[activity];
      const formattedDuration = formatDuration(duration);

      // 퍼센트 계산
      const percentage = totalDayMinutes > 0 ? Math.round((duration / totalDayMinutes) * 100) : 0;

      // 평균 에너지 계산
      let avgEnergy = "-";
      if (activityEnergies[activity] && activityEnergies[activity].totalDuration > 0) {
        // 해당 활동의 총 에너지 가중 합계를 총 시간으로 나누어 평균 계산
        const energyAvg = Math.round(activityEnergies[activity].totalEnergy / activityEnergies[activity].totalDuration);
        avgEnergy = `${energyAvg}`;
      }

      tableData.push([
        activity,
        formattedDuration,
        `${percentage}%`,
        avgEnergy
      ]);
    }

    // 테이블 데이터 정렬 (시간 기준 내림차순)
    tableData.sort((a, b) => {
      const getMinutes = (timeStr) => {
        if (timeStr === "-" || timeStr === "0m") return 0;
        let totalMinutes = 0;
        const hourMatch = timeStr.match(/(\d+)h/);
        const minuteMatch = timeStr.match(/(\d+)m/);
        if (hourMatch) totalMinutes += parseInt(hourMatch[1]) * 60;
        if (minuteMatch) totalMinutes += parseInt(minuteMatch[1]);
        return totalMinutes;
      };
      // 시간은 두 번째 열(index 1)
      return getMinutes(b[1]) - getMinutes(a[1]);
    });

    // 테이블 Markdown 생성
    let table = `| ${headers.join(" | ")} |\n`;
    table += `| ${headers.map(_ => "---").join(" | ")} |\n`;
    for (const row of tableData) {
      table += `| ${row.join(" | ")} |\n`;
    }

    // 테이블 출력
    dv.paragraph(table);

    // --- 요약 정보 계산 및 출력 ---
    let totalEnergyWeighted = 0;
    let totalEnergyDuration = 0;

    // 해당 날짜의 전체 평균 에너지 계산
    for (const activity in activityEnergies) {
        totalEnergyWeighted += activityEnergies[activity].totalEnergy;
        totalEnergyDuration += activityEnergies[activity].totalDuration;
    }

    const overallAvgEnergy = totalEnergyDuration > 0 ?
      Math.round(totalEnergyWeighted / totalEnergyDuration) : 0;

    dv.paragraph(`**총 기록 시간:** ${formatDuration(totalDayMinutes)} | **평균 에너지:** ${overallAvgEnergy}`);
  }
}
