핵심 메시지

Prompt가 아니라 계약

좋은 요청은 긴 문장이 아니라 Goal, Context, Constraints, Done when을 갖춘 실행 계약이다.

Plan before edit

복잡하거나 위험한 작업은 바로 수정하지 않고 ambiguity, assumption, validation을 먼저 나눈다.

Review first

사람은 모든 라인을 처음부터 읽기보다 Codex가 찾은 risk, missing test, decision point를 본다.

Evidence 연결

test pass를 완료 주장으로 바로 바꾸지 않고, 어떤 claim을 어떤 evidence가 지지하는지 확인한다.

반복은 Skill로

2~3회 반복 성공한 workflow는 prompt 모음이 아니라 skill, rule, guide로 승격한다.

자산을 남긴다

2개월 rollout의 성공 기준은 사용량보다 guide, FAQ, rule, skill, evidence pattern이 남는 것이다.

HW/IP 적용 방향

Spec/RTL agent 구현은 가능성이 있지만 난도가 높다. 첫 적용은 검증, 리뷰, evidence 정리부터 잡는다.

첫 메시지

지금 단계에서 세미나의 신뢰 포인트는 "AI가 RTL을 자동 생성한다"가 아니다. 실제 엔지니어가 바로 써볼 수 있는 read-heavy triage, review, evidence 정리, 반복 실수 지식화부터 보여주는 것이다.

Verification-first

Lint, compile, simulation, formal log를 분류하고 first evidence와 safe next check를 분리한다.

Decision boundary

Interface, protocol, PPA intent, product behavior는 사람이 잠그고 내부 detail은 option으로 둔다.

SSOT to RTL

Requirement, assumption, interface, parameter, open decision을 기준으로 RTL candidate를 만든다.

Ontology layer

SoC, subsystem, IPBlock, interface, requirement, verification case, evidence, owner를 연결한다.
